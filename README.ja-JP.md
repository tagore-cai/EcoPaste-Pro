## 機能紹介

- 🎉 Tauri v2 をもとに開発、軽量で効率的、クロスプラットフォームの体験をさらに向上させる。
- 💻 Windows、macOS、Linux（x11）とも対応でき、複数デバイス間でシームレスに切り替え可能である。
- ✨ シンプルで直感的なユーザーインターフェース、敷居が低く簡単に利用可能になる。
- 📋 テキスト、リッチテキスト、HTML、画像、ファイル形式のクリップボード内容をサポートできる。
- 🔒 データはローカルに保存され、ユーザーのプライバシーを保護、データはユーザー自身が完全に管理できる。
- 📝 メモ機能をサポートでき、簡単に分類、管理、検索が可能で作業効率を向上させる。
- ⚙️ 豊富なカスタマイズ設定で、異なるユーザーのニーズを満たす個別体験を提供できる。
- 🤝 完善なドキュメントとコミュニティ機能をサポート、開発者と共に成長を目指す。
- 🧩 継続的な最適化し、もっと驚きの機能があなたの発見を待っている。

## 🚀 Fork 版の更新履歴

> 本リポジトリは [EcoPasteHub/EcoPaste](https://github.com/EcoPasteHub/EcoPaste) の Fork 版であり、オリジナル版に基づき以下のユーザビリティ改善及び更新履歴を追加しています：

### Pro.5.x <font size="-2" color="gray">（元の v0.6.0-beta.3 ブランチに基づく）</font>

#### Pro.5.2

##### ✨ 新機能
- **📦 ローカルデータの絞り込みエクスポート**: ローカルデータのエクスポートエンジンを刷新し、「範囲 (すべて / お気に入りのみ) + 12 種類のコンテンツタイプ」を組み合わせた強力な絞り込みエクスポート機能を追加しました。エクスポートされたバックアップはフィルタ条件に一致したコンテンツおよび関連画像のみを含み、インポート・復元とも直接互換性があるため、より正確で実用的なバックアップが可能になります。
- **🏷️ バックアップの命名規則を統一**: WebDAV とローカルエクスポートにおけるバックアップファイルの命名規則を統一し（`アプリ名.タイムスタンプ.デバイス名.システム.モード.拡張子`）、`full`（完全）、`lite`（軽量）、`filter`（絞り込み）、`favs`（お気に入りのみ）の 4 つのバックアップモードを定義しました。バックアップファイルの管理がより明瞭になります。
- **📥 バックアップファイルのドラッグ＆ドロップインポート**: ローカルバックアップファイルをドラッグ＆ドロップしてデータを復元する機能を追加しました。インポート成功後、手動でアプリを再起動することなくクリップボードの履歴リストが自動的に更新されるため、データ復元が一層シームレスになります。
- **⌨️ 修飾キーのダブルクリックによる呼び出し**: 従来のショートカットキーの制限を打ち破り、「修飾キー（`Ctrl` / `Alt` / `Shift`）のダブルクリック」でクリップボードウィンドウを素早く開く機能を追加しました。設定パネルのショートカットキー設定に「記録 / ダブルクリック」の切り替えスイッチを追加し、`空間的な組み合わせキー`と`時間的なダブルクリック`を明確に分離することで、より直感的な設定とスムーズな操作を実現しました。

##### 💫 体験の最適化
- **✂️ 余白の自動削除**: メールアドレス、リンク、パス、カラーコードなどをコピーする際、誤ってコピーされた先頭や末尾の余計な空白を自動的に削除します。これにより、コンテンツが誤って単なる `プレーンテキスト` と判定されるのを防ぎ、タイプの識別精度を保証します。
- **🎨 コードのシンタックスハイライトを再構築**: コードのハイライト効果をアップグレードし、VS Code と同等のプロフェッショナルな配色を採用したことで、コードの可読性が大幅に向上しました。さらに視覚効果を最適化し、コードブロックとコンテンツカードがより自然に融合し、テキストに集中しやすくなりました。
- **🔄 インポート・復元後のシームレスなリロード**: `バックアップファイルのインポート` および `WebDAV からの復元` におけるデータ読み込みプロセスを根本から再構築しました。インポート後、アプリを再起動することなくクリップボードの履歴リストが即座に更新されるシームレスなホットリロード体験を実現しました。
- **🛡️ クリップボードの実際の変更を正確に識別**: バックグラウンドの監視ロジックを最適化し、サードパーティ製ソフトウェアや OS によって引き起こされる「偽のコピー」イベントを正確にフィルタリングします。クリップボードの内容が実際に変更された場合にのみ応答するようになり、バックグラウンドでの動作がより正確かつ効率的になりました。
- **🔍 同一内容をコピーした際のソース識別の最適化**: 異なるアプリで同じテキストや画像を連続してコピーした際、コンテンツのソースアプリアイコンと名前が正確に更新され、履歴がリストの一番上に正常にソートされるようになりました。より直感的な動作を実現しています。
- **🔒 履歴のソースの正確性を保護**: `自動ソート` モードにおいて、ペースト操作が履歴のソースアプリを誤って変更してしまう問題を修正しました。記録されたソース情報が正確であることを保証します。
- **📊 表の枠線を明確に表示**: Excel などのアプリから表のコンテンツをコピーする際、安全かつネイティブで他のスタイルと競合しない基本的なグリッド線スタイルを追加しました。これにより、データの構造と可読性が大幅に向上しました。
- **✂️ HTML コンテンツの表示効果の最適化**: 特定の HTML コンテンツをコピーした際、リストカードの上下に不要な余白が大きく表示される問題を修正しました。無駄な余白を自動的にトリミングし、よりコンパクトで美しいレイアウトを提供します。

##### 🐞 バグ修正
- **📧 メールのグループが空になる表示バグの修正**: `メール` グループを選択した際、正しく分類されているはずのメールの履歴が表示されない問題を修正しました。
- **📌 ピン留めウィンドウが貼り付け後に異常終了する問題の修正**: ピン留め（常に手前に表示）状態のクリップボードウィンドウで、各種貼り付け操作を実行した後にウィンドウが強制終了されてしまう問題を根本的に修正しました。ピン留めウィンドウが安定して表示されるようになります。

#### M05.1

##### ✨ 新機能（全く新しいストレージ管理体験）
- **📊 パノラマチャートビュー**
  - **📋 多次元ストレージ統計パネル**: 12の主要なコンテンツタイプのストレージ統計を追加し、スペースの占有状況を直感的に表示します。「すべて/お気に入り」の独立した切り替えをサポートし、お気に入りレコードの占有データを個別に確認できます。
  - **⚡ 超高性能統計エンジン**: 物理ボリュームのトレースと全体の統計アルゴリズムをリファクタリングし、ブロッキングディスク IO の走査を削除し、メモリレベルのデータベースクエリに統一しました。データの数万件、数百 GB であっても瞬時にロードできます。
  - **📈 動的視覚的バランスアルゴリズム**: 大小の差が激しい「ロングテールデータ」に対して、平方根の平滑化マッピング式を採用し、直感的なコントラストを兼ね備える一方で、小さいボリュームのデータが視覚的に「隠れる」のを避けます。
  - **🎯 パノラマデータ透過インタラクション**: 棒グラフのインタラクション領域を最適化し、マウスをホバーさせるとフローティングカードがポップアップ表示され、対応するタイプのアイテム数と正確なボリュームをリアルタイムで表示します。
- **⏳ 柔軟なタイムライン遷移**
  - **⏰ グローバル時間スマートフィルタリング**: 時間フィルターを内蔵し、`今日`、`昨日`、`過去3日間`、`今週`、`今月`、および`カスタム`の日付範囲をサポートし、指定期間のストレージデータをすばやく透視できます。
  - **🔗 全モジュール連動レスポンス**: 時間フィルターと統計チャート、詳細リストがシームレスに同期し、時間を切り替えるだけでデータの蓄積と変化の傾向を直感的に確認できます。
- **🧹 ご削除防止のスマートクリーンナップ**
  - **🗑️ タイプ別精密クリーンアップ**: コンテンツタイプ別の複数選択をサポートし、`未お気に入り` / `お気に入り` の不要な履歴データをワンクリックで一括クリーンアップします。
  - **🖼️ 画像の深い連動クリーンアップ**: 画像をクリーンアップする際、データベースのフィンガープリントを消去するだけでなく、「ローカル画像ファイルの同期削除」を同時にサポートし、ディスクスペースを真に解放し、カスタムストレージパスと完全に適合します。

##### 🐞 バグ修正
- **🌐 Webランクリッンク認識の修正**: クリップボードタイプの判定ロジックを最適化し、プレーンテキスト形式の URL/メールアドレスはプレーンテキストとして強制的に認識されるようになり、Web ページからコピーされたスタイル付きリンクが HTML として誤判定される問題を解決しました。
- **🎨 カラーコードの誤判定の修正**: 色認識の検証エンジンをリファクタリングし、標準のカラー値のみを対象とすることで、通常の英単語（例：`Tan`、`azure` など）が誤ってハイライトされるのを防ぎます。

##### 🐛 オリジナル版バグ修正
- **📂 ファイルアイコンパス分離の修正**: tauri-plugin-fs-pro の基盤 API を深く統合し、ファイルアイコンはユーザーのカスタムディレクトリに正確に保存され、システムのデフォルト `AppData` パスに残存しなくなりました。

#### M05.0

#### ✨ 新機能
- **📅 多次元組み合わせフィルタリング**：「日付 + コンテンツタイプ」の多次元複合フィルタリング機能を追加。上部のファネルアイコンをクリックしてフィルタパネルを呼び出せます。日別、月別、カスタム日付範囲の3つの独立した日付フィルタモードに対応し、同時に12種類のコンテンツタイプタグの複数選択をサポートし、効率的で正確なコンテンツ検査を実現します。
- **🧩 コード (SVG) スマート認識**：コードレベルでの SVG フォーマットコンテンツのスマート認識を追加。「正規表現によるコンポーネントスケルトンの事前スクリーニング + DOMParser による XML 構造解析」の二重検証でセキュリティを確保し、SVG 認識結果を明確な `コード (SVG)` ラベルとして分類表示します。
- **📌 詳細レベルのグローバル Tooltip**：コンテンツ項目のヘッダーの任意のテキスト領域にマウスを置くと、ソースアプリ、コンテンツタイプ、文字数（ファイルサイズ / 解像度）、作成時間などのコア属性を詳細に参照でき、より便利な詳細確認が可能です。

#### 💫 UX の最適化
- **📝 精密な時間適応表示**：クリップボード履歴の時間表示ロジックをリファクタリング。相対時間（例：「3日前」）が3日を超えると、自動的に `YY/M/D H:mm` 形式の絶対時間に切り替わり、時間の追跡がより明確で参考になります。
- **🧹 上部コントロールのレイアウト再構成**：検索フローティングウィンドウの上部レイアウトを最適化し、検索入力欄の幅を適切に狭め、「ピン留め / ピン解除」「環境設定」のアクションボタングループを前方に統合し、高頻度操作のアクセス効率を大幅に向上させました。
- **🏗️ フォーカス管理の完全ロールバック**：「フォーカスを奪わない静音ウィンドウ」「ウィンドウのカーソル追従」の基盤実装をオリジナル版 v0.6.0-beta.3 に完全ロールバック。Win32 API に強く依存するバックグラウンドポーリングスレッドと追加マウントインターフェースを分離し、メインウィンドウのネイティブフォーカスキャプチャメカニズムを復元しました。

#### 🐞 バグ修正
- **🚀 起動時のグループ切り替え無効**：「環境設定 → クリップボード → ウィンドウ呼び出し時に全グループに切り替え」の重要な設定が有効にならない問題を完全に解決し、アプリの起動時にユーザー設定のグループルールを厳密に遵守するようにしました。
- **📰 複合型コードの誤判定**：IDE 開発環境やウェブページから「プレーンテキストコード + HTML リッチテキストセマンティクス」の混合フラグメントをコピーした際に、コンテンツが `HTML` ハイパーテキストタイプとして強制認識される優先度オーバーライドの脆弱性を完全に修正し、コピーされたコードの正確な認識と分類を確保しました。
- **🌐 HTML コードの誤判定**：HTML の基盤判定ロジックをリファクタリングし、宣言ヘッダーや大文字小文字を区別しないシナリオの正規表現互換性を補完。スケルトン検証の失敗による誤判定を完全に解決し、認識精度を大幅に向上させました。
- **📝 Markdown の誤判定**：Markdown 検出ロジックを完全にリファクタリングし、「コア構文の正方向加重 + 不正特徴の負方向ペナルティ + 厳格なプレフィックス境界検証」の複合型インテリジェント判定エンジンにアップグレード。JS などの他のコードスニペットが Markdown として誤認識される問題を効果的に解決しました。
- **📋 JavaScript コードの誤判定**：基盤コード言語検出エンジンをリファクタリングし、JavaScript の判定優先度を最高に引き上げ、フロントエンドビルド成果物の強力な特徴キーワードを補充。同時に、JS のシフト演算と衝突する C++ 検出ルールの汎用演算子（`<<`、`>>`）を除去し、JS 特徴語の「リバースキル」検証メカニズムを追加。難読化・圧縮されたフロントエンド JS プロダクションコードが「コード (C++)」として誤認識される頑固な問題を完全に解決しました。
- **🗑️ レガシー残留ファイルの一掃**：廃止されたマウントに残存する一時ファイルを徹底的に比較・識別・クリーンアップし、冗長データによる今後の機能リファクタリングへの干渉を根本から排除しました。

#### 🐛 オリジナル版バグ修正
- **👁️ 項目ヘッダーの全幅解放**：オリジナル版 UI で右側の非表示操作ボタン用に強制確保されていた固定幅の「領域占有」動作を解除。未選択状態の記録ヘッダー属性バーが 100% 全幅表示を実現し、コンテンツ表示スペースがより充実し、情報表示がより完全になりました。

### M04.x <font size="-2" color="gray">（オリジナル版v0.6.0-beta.3ブランチに基づく）</font>
