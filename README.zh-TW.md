## 功能介紹

- 🎉 基於 Tauri v2 開發，輕量高效，跨平臺體驗更進一步。
- 💻 支持 Windows、macOS 和 Linux（x11），在多設備間無縫切換。
- ✨ 簡潔直觀的使用者介面，操作簡單，零門檻上手，開箱即用。
- 📋 支持純文字、富文字、HTML、圖片和檔案類型的剪貼板內容。
- 🔒 數據本地存儲，確保用戶隱私安全，數據完全掌控在用戶手中。
- 📝 支持備註功能，輕鬆分類、管理和檢索，讓工作更高效。
- ⚙️ 豐富的個性化設定，滿足不同用戶需求，打造專屬體驗。
- 🤝 完善的檔案與社區支持，與開發者共同探索與成長。
- 🧩 持續優化中，更多驚喜功能等你發現。

## 🚀 Fork 版本更新歷史

> 本倉庫是 [EcoPasteHub/EcoPaste](https://github.com/EcoPasteHub/EcoPaste) 的 Fork 版本，在原版基礎上增加了以下易用性改進與更新歷史：

### Pro.5.x <font size="-2" color="gray">（基於原版v0.6.0-beta.3分支）</font>

#### Pro.5.3

##### ✨ 新增功能
- **🚀 貼上為純文字升級全域快捷鍵**：重構「貼上為純文字」快捷鍵能力，打破僅限界內使用的侷限，正式升級為全域快捷鍵。軟體後台時一鍵貼上剪貼簿最新內容，前台時貼上選取內容，使用更靈活。

##### 💫 體驗優化
- **📊 重構剪貼簿類型識別引擎**：修復了因開啟「複製為純文字」模式後，從辦公軟體（Excel/WPS）複製資料被誤存為圖片的問題。
- **🎯 複合型表格資料智慧探測**：針對從辦公軟體（Excel/WPS）複製資料與瀏覽器複製圖片時極其相似的內容特徵，引入了四維探測體系，實現了對辦公表格資料的絕對精準定位與純淨化提取。
- **🧹 自動清理表格暫存檔案**：識別表格後，自動刪除無用暫存圖片，避免磁碟產生殘留檔案。
- **🔘 優化預設快捷按鈕**：預設開啟 9 項常用快捷操作並調整排序（偏好設定 → 剪貼簿 → 操作按鈕），全方位提升新使用者開箱即用的便捷體驗。
- **🖱️ 劃詞貼上功能優化**：將原「文字片段選取」功能更名為「劃詞貼上」（偏好設定 → 剪貼簿 → 操作按鈕），功能意圖表達更為直觀；同時在偏好設定中將其位置移動至「自動貼上」下方，設定時更直觀。
- **📊 儲存統計視圖空間優化**：深度精簡儲存統計頁面冗餘的包裹留白，左側統計摘要區字號與寬度動態縮窄，為橫向統計圖表提供最大化視覺化空間。

##### 🐞 bug 修復
- **⌨️ 修復貼上為純文字快捷鍵失效問題**：深度重構底層系統級按鍵注入邏輯，解決「貼上為純文字」和「快速貼上」時按鍵異常導致的無響應問題，貼上操作更穩定。
- **📈 修復統計圖表邊緣溢出**：徹底修復在極窄視窗尺寸下，統計圖表右側內容被強行擠出視窗外的佈局溢出問題。
- **⏰ 儲存統計時間選擇器自適應優化**：修復了在選擇「自訂」時間範圍時，右側超長的日期區間選擇框被強行擠出視窗外的佈局溢出問題。

#### Pro.5.2

##### ✨ 新增功能
- **📦 本地資料篩選匯出**：全新重構本地資料匯出引擎，新增「範圍 (全部 / 僅收藏) + 12 種內容類型」的多維組合篩選匯出能力。匯出的備份檔案僅包含匹配的內容及關聯圖片，且能直接兼容匯入還原，備份更精準、更實用。
- **🏷️ 全局標準化備份命名體系**：統一了 WebDAV 和本地匯出的備份檔案命名規則（`應用程式名.時間戳.設備名.系統.模式.副檔名`），並劃分出`full`（完整）、`lite`（輕量）、`filter`（篩選）、`favs`（僅收藏） 4 種備份模式，備份檔案管理更清晰。
- **📥 拖曳匯入備份檔案**：新增拖曳本地備份檔案還原資料的功能，匯入成功後自動重整剪貼簿清單，無須手動重啟應用程式，還原資料更便捷。
- **⌨️ 雙擊修飾鍵快捷喚起**：打破傳統組合鍵限制，新增「雙擊修飾鍵（`Ctrl` / `Alt` / `Shift`）」快捷打開剪貼簿視窗，設定面板新增「錄製 / 雙擊」切換開關，明確隔離`空間組合鍵`與`時間雙擊鍵`，配置更直觀、操作更順手。

##### 💫 體驗優化
- **✂️ 智能剔除邊緣空白**：複製信箱、連結、路徑或顏色值時，自動去掉內容首尾不小心複製的多餘空白，避免內容被誤判為`純文字`，保證類型識別精準。
- **🎨 程式碼語法高亮重構**：升級程式碼高亮效果，採用和 VS Code 一致的專業配色方案，程式碼可讀性大幅提升；同時優化視覺效果，程式碼區塊與內容卡片融合更自然，閱讀更沉浸。
- **🔄 匯入還原無感重載**：重構`匯入備份檔案`與`從 WebDAV 還原`的資料加載機制，實現匯入後無須重啟應用程式，剪貼簿歷史清單即時重整的無感熱重載體驗。
- **🛡️ 精準識別剪貼簿真實變化**：優化後台監聽邏輯，能精準過濾第三方軟體或系統導致的`假複製`事件，只響應真實的剪貼簿內容變更，後台運行更精準、更高效。
- **🔍 優化相同內容複製的來源識別**：連續在不同應用程式複製相同的文字 / 圖片時，能精準更新內容的來源應用程式圖示和名稱，且內容會正常置頂排序，符合使用直覺。
- **🔒 保護歷史紀錄來源準確性**：修復`自動排序`模式下，貼上操作會錯誤修改歷史紀錄來源應用程式的問題，確保紀錄的來源資訊準確無誤。
- **📊 清晰顯示表格框線**：從 Excel 等應用程式複製表格內容時，新增了安全、原生且不衝突的底層網格線保底樣式，極大增強了資料呈現的結構性和可讀性。
- **✂️ 優化 HTML 內容顯示效果**：修復複製 HTML 內容時，清單卡片首尾出現大片空白的問題，自動修剪多餘空白，顯示更緊湊美觀。

##### 🐞 bug 修復
- **📧 信箱分組顯示空白修復**：修復點擊`信箱`分組後，實際已正確分類的信箱紀錄無法顯示的問題。
- **📌 置頂視窗貼上後異常關閉修復**：徹底修復置頂狀態的剪貼簿視窗，在執行各類貼上操作後被強制關閉的問題，確保置頂視窗穩固顯示。

#### M05.1

##### ✨ 新增功能（全新儲存空間管家體驗）
- **📊 全景圖表視界**
  - **📋 多維儲存統計面板**：新增 12 大核心內容類型儲存統計，直觀展示空間佔用情況；支援「全部/收藏」獨立切換，可單獨查看收藏記錄佔用資料。
  - **⚡ 超高效能統計引擎**：重構物理體積溯源與大盤統計算法，移除阻塞性磁碟 IO 遍歷，統一改為記憶體級資料庫查詢，上萬條記錄、數百 GB 資料也能瞬時載入。
  - **📈 動態視覺平衡演算法**：針對大小懸殊的「長尾資料」，採用開方平滑映射公式，兼顧直觀對比的同時，避免小體積資料視覺「隱身」。
  - **🎯 全景資料透傳互動**：優化柱狀圖互動區域，滑鼠懸停即可彈出懸浮卡片，即時顯示對應類型的條目數量與精確體積。
- **⏳ 靈活的時間線流轉**
  - **⏰ 全域時間智慧過濾**：內建時間篩選器，支援`今天`、`昨天`、`近三天`、`本週`、`本月`及`自訂`日期範圍，快速透視指定時段儲存資料。
  - **🔗 全模組聯動響應**：時間篩選與統計圖表、詳情列表無縫同步，切換時間即可直觀查看資料堆積與變化趨勢。
- **🧹 防誤刪的智慧大掃除**
  - **🗑️ 按類型精準清理**：支援按內容類型多選，一鍵批量清理 `未收藏` / `已收藏` 的冗餘歷史資料。
  - **🖼️ 圖片深度聯動清理**：清理圖片時，系統除了抹除資料庫指紋，同時支援「同步刪除本地圖片檔案」，真正釋放磁碟空間，完美適配自訂儲存路徑。

##### 🐞 Bug 修復
- **🌐 網頁連結識別修復**：優化剪貼簿類型判斷邏輯，純文字格式的URL/郵箱會強制識別為純文字，解決從網頁複製帶樣式連結被誤判為HTML的問題。
- **🎨 顏色代碼誤判修復**：重構顏色識別校驗引擎，僅標準色值，避免普通英文單詞（如 `Tan`、`azure` 等）被誤判高亮。

##### 🐛 原版缺陷修復
- **📂 檔案圖示路徑隔離修復**：深度對接 tauri-plugin-fs-pro 底層 API，檔案圖示將精準儲存至使用者自訂目錄，不再殘留到系統預設 `AppData` 路徑。

#### M05.0

#### ✨ 新增功能
- **📅 多維組合篩選**：新增「日期 + 內容類型」多維度複合篩選能力，點擊頂部漏斗圖示即可喚起篩選面板。支援按日、按月、自訂時間範圍三種獨立日期篩選模式，同時支援 12 種內容類型標籤多選，打造高效精準的內容檢視鏈路。
- **🧩 代碼 (SVG) 智慧識別**：新增代碼層面的 SVG 格式化內容智慧識別，依託「正則初篩元件骨架 + DOMParser 解析 XML 結構」雙重校驗構建安全壁壘，並將 SVG 識別結果歸類展示為明確的 `代碼 (SVG)` 標籤。
- **📌 詳情級全域 Tooltip**：滑鼠懸停在內容條目頭部任意文字區域，即可詳盡追溯內容全量核心屬性，涵蓋來源、內容類型、字元數（檔案大小 / 解析度）、建立時間等資訊，詳情查看更便捷。

#### 💫 體驗優化
- **📝 精準時間自適應顯示**：重構剪貼簿歷史記錄時間顯示邏輯，當相對時間（如「3 天前」）跨度超過 3 天時，將自動切換為 `YY/M/D H:mm` 格式的絕對時間，時間追溯更清晰、更具參考性。
- **🧹 頂部控制元件佈局重組**：優化搜尋懸浮窗頂部佈局，合理收窄搜尋輸入框視覺寬度，同時將「置頂 / 取消置頂」「偏好設定」功能按鈕組前置整合，大幅提升高頻操作的觸達效率。
- **🏗️ 焦點管理全鏈路回退**：「不奪焦靜默視窗」「視窗跟隨游標位置」底層實現全量回退至原版 v0.6.0-beta.3 版本，剝離強依賴 Win32 API 的後臺輪詢執行緒與額外掛載介面，恢復主視窗原生焦點捕獲機制。

#### 🐞 bug 修復
- **🚀 啟動時分組切換失效**：徹底解決「偏好設定 - 剪貼簿 - 視窗呼出時切換至全部分組」關鍵配置失效的問題，確保應用啟動時嚴格遵循使用者設定的分組規則。
- **📰 複合型代碼誤判**：徹底修復從 IDE 開發環境或網頁中複製同時攜帶「純文字代碼 + HTML 富文字語義」混合片段時，內容被粗暴定性進而被強制識別為 `HTML` 超文字類型的底層優先級越權漏洞，確保複製的代碼內容可被精準識別歸類。
- **🌐 HTML 代碼誤判**：重構 HTML 代碼底層判定邏輯，補全對聲明頭部、大小寫不敏感場景的正則相容支援，徹底解決因骨架校驗失效導致的內容誤判問題，識別準確率大幅提升。
- **📝 Markdown 誤判**：徹底重構 Markdown 檢測邏輯，升級為「核心語法正向加權 + 非法特徵負向懲罰 + 嚴格前綴邊界校驗」的複合型智慧判定引擎，有效解決 JS 等其他代碼片段被誤判為 Markdown 格式的問題。
- **📋 JavaScript 代碼誤判**：重構底層代碼語言檢測引擎，將 JavaScript 判定優先級調至最高，針對性補充前端建構編譯產物的強特徵詞；同時移除 C++ 檢測規則中易與 JS 位移運算衝突的通用運算子（`<<`、`>>`），新增 JS 特徵詞「反向查殺」校驗機制，徹底解決混淆、壓縮後的前端 JS 生產代碼被誤識別為「代碼 (C++)」的頑固問題。
- **🗑️ 歷史遺留垃圾清剿**：深度比對甄別並徹底清理廢棄掛載的殘存臨時檔案，從根源杜絕冗餘資料對後續功能重構的干擾。

#### 🐛 原版缺陷修復
